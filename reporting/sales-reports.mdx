## Sales Out Report

**Location:** Reports > Sales Out Report

**Purpose:** Analyze product-level sales performance showing sales quantities, sales amounts, returns, and net sales revenue.

### Key Metrics

<AccordionGroup>
  <Accordion title="Sales Performance Indicators">
    - **Sales Quantity** (销售数量): Total units sold
    - **Sales Amount** (销售金额): Total revenue from sales
    - **Return Quantity** (退货数量): Units returned by customers
    - **Return Amount** (退货金额): Revenue lost to returns
    - **Net Sales Amount** (实际销售金额): Sales Amount - Return Amount
  </Accordion>

  <Accordion title="Product Information">
    - Barcode, name, specifications, model
    - Color, brand, manufacturer
    - Custom property fields (3 configurable fields)
    - Unit of measure
  </Accordion>

  <Accordion title="Multi-Dimensional Filtering">
    - Date range (document date)
    - Customer selection
    - Warehouse/depot
    - Organization/department
    - Material category
  </Accordion>
</AccordionGroup>

### How to Use

1. **Access the Report**
   - Navigate to **Reports > Sales Out Report**
   - Report loads with default date range (last 3 months)

2. **Set Date Range** (Required)
   ```
   Document Date: Select start and end dates
   Default: Previous 3 months to today
   This filter is mandatory for performance
   ```

3. **Apply Additional Filters** (Optional)
   ```
   Material Info: Search by barcode, name, specs, model
   Customer: Select specific customer
   Warehouse: Filter by warehouse location
   Organization: Filter by department/branch
   Category: Choose material category
   ```

4. **Analyze Results**
   - View **Net Sales Amount** at top of page
   - Sort by sales amount to identify top products
   - Review return quantities to spot quality issues
   - Compare performance across different products

5. **Export and Share**
   - Click **Export** for Excel download
   - Share with sales team for analysis
   - Use data for sales planning

<Note>
  The report displays the Net Sales Amount summary at the top, which equals total sales minus total returns. This is your actual realized revenue for the period.
</Note>

**Purpose:** View detailed line-item information for all outbound transactions including sales orders, retail sales, and other shipments.

<AccordionGroup>
  <Accordion title="Transaction Details">
    - Document number and type
    - Transaction date
    - Customer/organization
    - Warehouse/depot
    - Approval status
  </Accordion>

  <Accordion title="Line Item Details">
    - Material information (barcode, name, specs)
    - Quantity shipped
    - Unit price
    - Line amount
    - Tax information
    - Discount applied
  </Accordion>

  <Accordion title="Search and Filter">
    - Date range selection
    - Customer filter
    - Material search
    - Document number search
    - Status filter
  </Accordion>
</AccordionGroup>

### How to Use

1. **Access Detail Report**
   - Navigate to **Reports > Out Detail**
   - Report shows all outbound transactions

2. **Filter Transactions**
   - Select date range
   - Choose customer or warehouse
   - Search by document number
   - Filter by material

3. **Review Line Items**
   - See every individual line item
   - Verify quantities and prices
   - Check approval status

4. **Export for Analysis**
   - Export to Excel for detailed analysis
   - Use for reconciliation
   - Share with auditors

### Use Cases

- **Transaction Verification**: Verify specific sales orders and shipments
- **Audit Trail**: Provide detailed transaction history for audits
- **Dispute Resolution**: Research customer inquiries about specific orders
- **Data Analysis**: Extract transaction data for advanced analytics

## Out Material Count

**Location:** Reports > Out Material Count

**Purpose:** Summarize total quantities shipped by material across all outbound transactions.

### What It Shows

- Material identification (barcode, name, specs)
- Total quantity shipped in period
- Number of transactions
- Summary by material category
- Aggregated across all customers and warehouses

### How to Use

1. **Access Summary Report**
   - Navigate to **Reports > Out Material Count**
   - Select date range

2. **Filter Results**
   - Choose material category
   - Search specific materials
   - Select warehouse

3. **Analyze Volume**
   - Identify high-volume materials
   - Plan inventory replenishment
   - Forecast future demand

### Use Cases

- **Demand Planning**: Analyze material demand patterns for forecasting
- **Inventory Planning**: Plan purchases based on shipment volumes
- **Capacity Planning**: Ensure adequate warehouse capacity for high-volume items
- **Performance Metrics**: Track throughput and shipment volumes

## Troubleshooting

<AccordionGroup>
  <Accordion title="Sales amounts don't match expectations">
    **Solutions:**
    - Verify all sales orders are approved and posted
    - Check if date range includes all relevant transactions
    - Ensure no orders are pending approval
    - Review if returns are properly recorded
    - Check for any data entry errors
  </Accordion>

  <Accordion title="Missing transactions in report">
    **Solutions:**
    - Verify transactions are within selected date range
    - Check that orders have been approved
    - Ensure warehouse filter includes correct locations
    - Clear customer filter if accidentally set
    - Verify transactions are not still in draft status
  </Accordion>

  <Accordion title="Return quantities seem incorrect">
    **Solutions:**
    - Verify all return orders are properly posted
    - Check that returns are linked to original sales
    - Review return document dates
    - Ensure return warehouse is correct
  </Accordion>

  <Accordion title="Export file is too large">
    **Solutions:**
    - Reduce date range
    - Filter by specific customer or category
    - Export in smaller batches
    - Contact system administrator about performance
  </Accordion>
</AccordionGroup>
