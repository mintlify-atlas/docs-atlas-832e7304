## Overview

The Depot API provides comprehensive endpoints for managing warehouses, inventory transactions, and stock movements in jshERP. This includes depot management, depot head (transaction headers), and depot items (transaction details).

## Depot Management

Manage warehouse/depot entities including creation, updates, and configuration.

### Get Depot Info

Retrieve detailed information about a specific depot by ID.

```bash
GET /depot/info?id={id}
```

<ParamField query="id" type="Long" required>
  The unique identifier of the depot
</ParamField>

<ResponseField name="code" type="integer">
  Response status code (200 for success)
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Depot object">
    <ResponseField name="id" type="Long">Depot ID</ResponseField>
    <ResponseField name="name" type="string">Depot name</ResponseField>
    <ResponseField name="address" type="string">Physical address</ResponseField>
    <ResponseField name="warehousing" type="BigDecimal">Warehousing fee</ResponseField>
    <ResponseField name="truckage" type="BigDecimal">Trucking fee</ResponseField>
    <ResponseField name="type" type="integer">Depot type</ResponseField>
    <ResponseField name="principal" type="Long">Principal user ID</ResponseField>
    <ResponseField name="enabled" type="boolean">Whether depot is enabled</ResponseField>
    <ResponseField name="isDefault" type="boolean">Whether this is the default depot</ResponseField>
    <ResponseField name="remark" type="string">Additional remarks</ResponseField>
  </Expandable>
</ResponseField>

### List Depots

Retrieve a paginated list of depots with optional search filters.

```bash
GET /depot/list?search={search}
```

<ParamField query="search" type="string">
  JSON string containing search parameters:
  - `name`: Depot name filter
  - `type`: Depot type filter
  - `remark`: Remark filter
</ParamField>

### Create Depot

Create a new depot/warehouse.

```bash
POST /depot/add
```

<ParamField body="name" type="string" required>
  Depot name
</ParamField>

<ParamField body="address" type="string">
  Physical address
</ParamField>

<ParamField body="type" type="integer">
  Depot type
</ParamField>

<ParamField body="principal" type="Long">
  Principal user ID
</ParamField>

<ParamField body="enabled" type="boolean">
  Enable/disable depot (default: true)
</ParamField>

<ParamField body="remark" type="string">
  Additional remarks
</ParamField>

<CodeGroup>
```bash cURL
curl -X POST "http://localhost:8080/depot/add" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Main Warehouse",
    "address": "123 Storage St, City",
    "type": 1,
    "enabled": true,
    "remark": "Primary storage facility"
  }'
```

```json Response
{
  "code": 200,
  "data": {
    "message": "操作成功"
  }
}
```
</CodeGroup>

## Depot Head (Transaction Headers)

Manage depot transaction headers including purchase orders, sales orders, transfers, and other inventory movements.

### Get Depot Head Info

```bash
GET /depotHead/info?id={id}
```

<ParamField query="id" type="Long" required>
  Depot head ID
</ParamField>

### List Depot Heads

Retrieve depot transaction headers with advanced filtering.

```bash
GET /depotHead/list?search={search}
```

<ParamField query="search" type="string">
  JSON string containing search parameters:
  - `type`: Transaction type (入库/出库/其它)
  - `subType`: Sub-type (采购/销售/调拨/etc.)
  - `number`: Bill number
  - `beginTime`: Start date
  - `endTime`: End date
  - `organId`: Organization ID
  - `depotId`: Depot ID
  - `status`: Status filter
</ParamField>

### Create Depot Head and Details

Create a new depot transaction with header and line items.

```bash
POST /depotHead/addDepotHeadAndDetail
```

<ParamField body="info" type="string" required>
  JSON string of depot head information
</ParamField>

<ParamField body="rows" type="string" required>
  JSON string of depot item rows
</ParamField>

<CodeGroup>
```bash cURL
curl -X POST "http://localhost:8080/depotHead/addDepotHeadAndDetail" \
  -H "Content-Type: application/json" \
  -d '{
    "info": "{\"type\":\"入库\",\"subType\":\"采购\",\"organId\":1,\"operTime\":\"2024-01-15 10:00:00\"}",
    "rows": "[{\"materialId\":101,\"operNumber\":50,\"unitPrice\":25.50,\"depotId\":1}]"
  }'
```

```json Response
{
  "code": 200,
  "data": {
    "message": "操作成功"
  }
}
```
</CodeGroup>
