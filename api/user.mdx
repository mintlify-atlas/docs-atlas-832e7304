## Overview

The User Management API provides endpoints for managing users, authentication, and user-related operations in jshERP. All endpoints require proper authentication unless otherwise specified.

**Base Path:** `/user`

---

## Authentication Endpoints

### User Login

<Note>This endpoint does not require authentication.</Note>

```http
POST /user/login
```

Authenticates a user with login credentials and captcha verification.

<ParamField body="loginName" type="string" required>
  User's login name
</ParamField>

<ParamField body="password" type="string" required>
  User's password (MD5 hashed)
</ParamField>

<ParamField body="code" type="string" required>
  Captcha verification code
</ParamField>

<ParamField body="uuid" type="string" required>
  UUID from captcha request
</ParamField>

<ResponseField name="code" type="integer">
  Response status code (200 for success, 500 for error)
</ResponseField>

<ResponseField name="data" type="object">
  Contains authentication token and user information
</ResponseField>

<CodeGroup>
```json Request Example
{
  "loginName": "admin",
  "password": "5f4dcc3b5aa765d61d8327deb882cf99",
  "code": "1234",
  "uuid": "a1b2c3d4e5f6g7h8"
}
```

```json Response Example
{
  "code": 200,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 1,
      "username": "Administrator",
      "loginName": "admin"
    }
  }
}
```
</CodeGroup>

### WeChat Login

```http
POST /user/weixinLogin
```

Authenticates a user using WeChat authorization code.

<ParamField body="weixinCode" type="string" required>
  WeChat authorization code
</ParamField>

<ResponseField name="code" type="integer">
  - `200`: Success
  - `501`: WeChat not bound
  - `500`: Login failed
</ResponseField>


---

### Get User List

```http
GET /user/list?search={search}
```

Retrieves a paginated list of users with optional filtering.

<ParamField query="search" type="string">
  JSON string containing search filters:
  - `userName`: Filter by display name
  - `loginName`: Filter by login name
</ParamField>

<ResponseField name="rows" type="array">
  Array of UserEx objects with extended information
</ResponseField>

<ResponseField name="total" type="integer">
  Total number of records
</ResponseField>

<CodeGroup>
```bash Request Example
curl -X GET "http://localhost:9999/jshERP-boot/user/list?search={\"userName\":\"John\"}" \
  -H "Authorization: Bearer {token}"
```

```json Response Example
{
  "rows": [
    {
      "id": 1,
      "username": "John Doe",
      "loginName": "john",
      "email": "john@example.com",
      "phonenum": "1234567890",
      "status": 1
    }
  ],
  "total": 1
}
```
</CodeGroup>

## Password Management

### Reset Password

```http
POST /user/resetPwd
```

Resets a user's password (admin function).

<ParamField body="id" type="long" required>
  User ID
</ParamField>

<ParamField body="password" type="string" required>
  New password (MD5 hashed)
</ParamField>

---

### Update Password

```http
PUT /user/updatePwd
```

Allows a user to change their own password.

<ParamField body="userId" type="long" required>
  User ID
</ParamField>

<ParamField body="oldpassword" type="string" required>
  Current password (MD5 hashed)
</ParamField>

<ParamField body="password" type="string" required>
  New password (MD5 hashed)
</ParamField>

<ResponseField name="status" type="integer">
  - `1`: Success
  - `2`: Old password incorrect
  - `3`: Update failed
</ResponseField>

## Utility Endpoints

### Check Login Name Exists

```http
GET /user/checkIsNameExist?id={id}&name={name}
```

Checks if a login name is already in use.

<ParamField query="id" type="long" required>
  User ID (0 for new user)
</ParamField>

<ParamField query="name" type="string">
  Login name to check
</ParamField>

<ResponseField name="status" type="boolean">
  `true` if name exists, `false` otherwise
</ResponseField>
